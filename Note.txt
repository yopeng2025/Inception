Virtual Machine:
    在物理宿主机（我的电脑physical host machine）上装一个独立的虚拟机（guest virtual machine），确保有完全隔离的，干净的操作系统环境

Docker(工厂): 
    打包代码+运行环境
    保证任何机器的运行结果一致
    解决“在我的电脑上能跑，在你的电脑上不行”的问题

    Container(容器):        轻量级的、独立的软件包，包含了运行某个软件所需的一切（代码、运行环境、系统工具等）
    Dockerfile(产品设计图):  告诉Docker如何从零开始一步步把这个环境搭建起来(制定具体版本号，不使用latest)
    Docker Compose(工程师): 如果你有多个容器（NGINX, MariaDB 等），他负责一键启动并管理它们

NGINX(TLS1.2/1.3):
    (门卫，翻译)整个架构的唯一入口。
    TLS (Transport Layer Security): 加密协议。在浏览器输入网址时，TLS 确保你和服务器之间的通讯是加密的（即 HTTPS），防止数据被窃听

WordPress:
    网站， 用PHP语言编写
    PHP-FPM负责解析PHP代码，生成网页内容发送给NGINX

MariaDB:
    存储数据的“仓库”，里面包含WordPress的文章，用户信息，配置

Volumes:
    容器重启后，内部数据会丢失，卷==外接硬盘，将数据存放在宿主机（home/login/data），确保数据不丢失

Network:
    让独立的容器之间能够相互通信(如，WordPress需要MariaDB提供数据)



具体操作
1. 环境准备
    安装Virtualbox,debian系统（software selection阶段选择[Debian desktop environment]+[GNOME]/[Xfce]可以直接使用GUI！）
    安装Docker，DockerCompose
    Makefile
2. 搭建结构
    srcs -> docker-compose.yml
    srcs -> requirements(nginx, mariadb, wordpress)
    /etc/hosts -> yopeng指向127.0.0.1(主机环回地址)
3. 编写Dockerfile
    MariaDB:  写Dockerfile安装MariaDB， 脚本->初始化用户和密码（禁止死循环）
    WordPress:下载WordPress，不能包含NGINX，通过端口9000与外界沟通
    NGINX:    配置SSL(Secure Sockets Layer)证书(openssl生成自签名证书)，设置配置文件只允许TLS1.2/1.3，将请求转发给WordPress容器
4. 整合
    编写docker-compose.yml，将所有服务连接在一起，配置Network和Volumes
    建立.env文件，储存敏感信息(数据库密码等，不要提交到Git)，并在Dockerfile中引用
5. 文档
    ReadMe.md 
    user_doc.md 
    dev_doc.md

Debian虚拟机中配置静态IP 以保持环境稳定
VM：
    ip a
    ip route | grep default
    网卡名 enp0s3
    IP 10.171.49.222
    Netmask /20
    Gateway 10.171.48.1(网段第1个地址)

SSH:
    sudo nano /etc/network/interfaces

“   auto lo
    iface lo inet loopback

    # 配置主网卡 enp0s3
    auto enp0s3
    iface enp0s3 inet static
        address 10.171.49.222
        netmask 255.255.240.0
        gateway 10.171.48.1
        dns-nameservers 8.8.8.8 1.1.1.1
”
    重启网络服务
    sudo systemctl restart networking
    ip a 检查ip是否是10.171.49.222
    ping yopeng.42.fr
    https://yopeng.42.fr
